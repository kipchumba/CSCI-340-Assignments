CSCI 340 Assignment 2

Purpose

This assignment builds on last week's assignment. It replaces the array and loop indices by the STL vector class and iterators. It also uses a command line argument to acquire the file name. The driver requires a copy constructor and an overloaded assignment operator to work properly.

Assignment

Make the following changes to your program from HW1.

Using the STL vector class and iterators

Replace the array in your player database by an STL vector of pointers to baseball players.

Where possible, use an iterator to loop through the vector.

Good team and small team

Your program will produce four output files. The first three include one for the original team, one for a team composed only of good players, and one for a small team. The fourth one prints the original team again. If you have coded your copy constructor and overloaded assignment operator correctly, the last output should be the same as the first.

Add the argument ofstream& outFile to PlayerDatabase::print_team to handle this. You will also have to make corresponding changes to the BaseballPlayer, Pitcher and Hitter classes.

Write a member function void create_good_team() to convert the team into a better one by removing all the players who are not considered good. We will define a good pitcher as one with an ERA less than or equal to 3.4950. We will define a good hitter as one with as a batting average greater than or equal to .2795.

Write a member function void create_small_team() to convert the team into a smaller one by deleting the first four players. Your code should work regardless of how many players are on the team: 0, less than 4, exactly 4, or more than 4.

Use suitable vector functions for each of these.

If there are no pitchers or hitters available to calculate a team average, print "n/a".

Remember that the elements of team are pointers, so you must delete the BaseballPlayer objects in addition to removing the undesired elements from the vector.

File name entered as command-line arguments

You will execute your program as

./hw2.exe xyz

where xyz is the prefix for your files.

The driver will call a function with signature

void create_names(int argc, char* argv[], string& iFileName,
    string& oFileNameAll, string& oFileNameGood, string& oFileNameSmall,
    string& oFileNameRepeat);
to create the names for all of the files.

If the input name is xyz, the input will be xyz.txt. Build the output names as xyz_output_all.txt, xyz_output_good.txt, xyz_output_small.txt and xyz_output_repeat.txt.

You can use the overloaded '+' operator to concatenate two strings.

Write this function and include create_names.h and create_names.cpp in your makefile.
